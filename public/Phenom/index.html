Problem Statement

Phenom is a purpose-driven, global enterprise SaaS company headquartered in Ambler, Pennsylvania, delivering AI-powered talent experience solutions to over 500 enterprises worldwide.

Their primary use case to date has been support and build operations. This PoC expands DevRev's footprint into deeper AI-powered support intelligence and conversational CX, integrating tightly with Phenom's existing ServiceNow stack through bidirectional ticket sync and Knowledge Base import — so the AI experience layers cleanly on top of Phenom's systems of record without disrupting them.

In Scope

Two Use Cases

This PoC focuses on demonstrating core value through two use cases, deployed on DevRev's playground using Phenom's existing ServiceNow data.

Use Case 1

Internal AI Assistance for Support Agents (EX Agent)

AI-powered tools to help Phenom's support agents resolve tickets faster through intelligent search, auto-classification, and response suggestions — grounded in the ingested ServiceNow KB and historical ticket data.

  • Auto-Triage (Pre-Agent) — the AI agent reads the full ticket content and automatically enriches it: setting the correct priority, applying relevant tags, identifying the product area, assessing urgency and customer sentiment, and routing it to the right group or agent.
  • Similar Ticket Matching — surface past resolved tickets with successful resolutions.
  • Response Suggestion — AI-generated resolution draft added to ticket notes automatically.
  • Knowledge Base Search — query KBs and policies for context inline.
  • Analytical Insights — AI-driven ticket clustering and advanced analytical queries to derive actionable intelligence and identify emerging support trends from historical data.
Use Case 2

CX Agent (Customer-Facing Chatbot)

Deploy Computer for Customers on DevRev's playground to handle common customer inquiries autonomously, reducing the volume of L1 tickets that reach the support team. Unlike traditional rule-based chatbots, Computer's CX Agent reasons through customer questions using Phenom's own knowledge base, documentation and historical ticket data — delivering accurate, context-aware answers in real time. When it can't resolve an issue, it hands off to a human agent with the full conversation context already attached, so agents never have to ask the customer to repeat themselves. Phenom's existing DevRev deployment means the CX Agent is grounded in real Phenom products and support data from day one, significantly reducing the typical ramp time.

Success Criteria

How We'll Measure Success

To determine if the scope of work is successful, Phenom will validate that DevRev was able to deliver on these four outcomes.

01

CX Deflection Rate

The chatbot handles FAQs, configuration guidance and access requests directly in Phenom's portal, resolving customer queries without human intervention.

Target: ≥30% deflection rate
02

Deflection Quality

The chatbot provides correct, relevant answers to customer queries — ensuring deflected tickets are genuinely resolved, not just bounced.

Target: ≥80% answer accuracy on deflected cases
03

Conversation-to-Ticket Context Preservation

When the CX Agent escalates a conversation to a human agent, the full interaction history is automatically attached to the created ticket.

Target: 100% of escalated conversations carry full context into the ticket
04

Similar Ticket Relevance

When agents receive a new ticket, Computer surfaces past resolved tickets with matching symptoms, product area or resolution pattern. Agents validate whether the suggestions are contextually relevant.

Target: ≥70% of similar-ticket suggestions rated "useful" by agents during the pilot
